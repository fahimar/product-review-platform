# ReviewDibo

A full-stack product review platform — browse products, read reviews, and submit your own.

**Stack:** FastAPI (async) · SQLAlchemy 2.0 · Alembic · PostgreSQL 16 · Next.js 15 (App Router) · TypeScript · Tailwind CSS · shadcn/ui

## Local setup

### Prerequisites

| Tool | Version |
|------|---------|
| Docker & Docker Compose | v24+ |
| Python | 3.12+ |
| [uv](https://github.com/astral-sh/uv) | latest |
| Node.js | 20+ |
| npm | any |

### Option A — full stack in Docker

```bash
# 1. Clone and enter the repo
git clone <repo-url> && cd product-review-platform

# 2. Start Postgres + backend
docker compose up --build -d

# 3. Run migrations and seed data
docker compose exec backend uv run alembic upgrade head
docker compose exec backend uv run python scripts/seed.py

# 4. Start the frontend
cd frontend
cp .env.example .env.local          # default: NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
npm install
npm run dev
```

### Option B — backend locally (faster iteration)

```bash
# Terminal 1 — Postgres only
docker compose up db -d

# Terminal 2 — backend
cd backend
cp .env.example .env
uv sync
uv run alembic upgrade head
uv run python scripts/seed.py
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Terminal 3 — frontend
cd frontend
cp .env.example .env.local
npm install
npm run dev
```

## Environment variables

### Backend (`backend/.env`)

```env
DATABASE_URL=postgresql+asyncpg://reviewdibo:reviewdibo@localhost:5432/reviewdibo
JWT_SECRET=change-me-to-a-long-random-secret
JWT_EXPIRE_MINUTES=60
CORS_ORIGINS=["http://localhost:3000"]
```

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | asyncpg connection string |
| `JWT_SECRET` | Secret key for JWT signing — use a long random string in production |
| `JWT_EXPIRE_MINUTES` | Token lifetime in minutes |
| `CORS_ORIGINS` | JSON array of allowed origins |

### Frontend (`frontend/.env.local`)

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

## Deployment

### 1 — Database: Neon (free Postgres)

1. Sign up at [neon.tech](https://neon.tech), create a project.
2. Copy the **connection string** — it looks like:
   ```
   postgresql+asyncpg://user:pass@ep-xxx.us-east-1.aws.neon.tech/neondb?sslmode=require
   ```
3. Set it as `DATABASE_URL` in the backend's env vars on Render (see below).

### 2 — Backend: Render (Blueprint / render.yaml)

The repo ships a `render.yaml` at the root. Render auto-detects it as a **Blueprint**.

**Steps:**
1. Go to [render.com/dashboard](https://render.com/dashboard) → **New → Blueprint**.
2. Connect this repository — Render reads `render.yaml` and creates the `reviewdibo-backend` web service automatically.
3. In the Render dashboard for the service, open **Environment** and set the two manually-managed variables:

   | Key | Value |
   |-----|-------|
   | `DATABASE_URL` | Neon asyncpg connection string (e.g. `postgresql+asyncpg://user:pass@ep-xxx.neon.tech/neondb?sslmode=require`) |
   | `CORS_ORIGINS` | Comma-separated list of allowed origins, e.g. `https://your-vercel-app.vercel.app` |

   `JWT_SECRET` is auto-generated by Render (declared with `generateValue: true` in `render.yaml`).

4. Click **Deploy** — Render runs:
   ```
   pip install -r requirements.txt          # build
   alembic upgrade head && uvicorn ...      # start (migrations run on every deploy)
   ```

5. After the first deploy, seed the database from the Render **Shell** tab:
   ```bash
   python scripts/seed.py
   ```

> **Free-tier note:** Render free services spin down after ~15 minutes of inactivity. The next request triggers a cold start that takes ~30–50 s. The `/health` endpoint (used as `healthCheckPath`) lets external uptime monitors keep the service warm.

### 3 — Frontend: Vercel

1. Import the repo in [vercel.com](https://vercel.com).
2. Set **Root Directory** to `frontend`.
3. **Environment variable:**

   | Key | Value |
   |-----|-------|
   | `NEXT_PUBLIC_API_BASE_URL` | `https://your-render-app.onrender.com` |

4. Deploy — Vercel auto-detects Next.js.
